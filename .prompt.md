# Prompt 模板：将用户问答记录并同步到 `doc/大数据面试题.md`

目的：
- 自动化把用户提出的面试问题（Question）以与现有 `doc/大数据面试题.md` 风格一致的 Markdown 条目追加到文档中，并为每个问题生成详尽、严谨、可复盘的答案（Answer）。

适用场景：
- 用户在对话中逐条提出大数据相关面试问题，期望把问题与答案记录到 `doc/大数据面试题.md`，文风需与原文档一致（结构化、条理清晰、包含示例与知识扩展）。

输入（必需）：
- `question`: 用户提出的问题文本（中文或英文）。
- `category`（可选）：问题分类，如 `Spark`、`Flink`、`存储` 等；若不填，自动尝试根据问题内容匹配或创建新分类。
- `format_requirements`（可选）：用户对回答的格式要求，例如是否需要代码示例、SQL 示例、复杂度分析、知识扩展等。默认遵循 `doc/大数据面试题.md` 的前端 prompt 要求。

行为规范：
1. 首先读取并引用 `doc/大数据面试题.md` 最前面的 prompt，确保遵循该 prompt 中的所有风格与要求（例如：使用 Markdown、英文括号、包含知识扩展等）。
2. 将 `question` 按照文档当前的结构（例如：`## 1. Spark 面试题` 下的 `### 1.1` 子条目样式）插入或追加：
   - 若 `category` 与现有章节匹配，则在对应章节下追加新的子条目（按顺序编号或使用新的二级/三级标题）。
   - 若找不到合适章节，则创建一个新的章节，标题格式为中文分类名称（例如：`## X. 自定义分类`）。
3. 为问题生成答案，要求：
   - 回答需严谨、有条理，包含触发条件、适用场景、实现/排查方法、示例（若适用则包含代码或 SQL 并带注释）、最后的知识扩展（列举相关主题并说明关联）。
   - 保持与文档现有条目的写作风格一致（语言简洁、结构化、面向面试）。
   - 对于复杂主题，给出 `排查/落地` 建议与常见误区。
4. 格式化输出为 Markdown，尽量复用文档中已有表格、代码块样式和小节划分。
5. 在文档中标注新增条目的时间戳与作者为 `assistant`（可选，遵循现有文档风格）。
6. 在每次追加后，返回给用户一个简短的确认信息：说明已在哪个章节追加了哪个问题，并提供文档链接。

输出（将写回 `doc/大数据面试题.md`）：
- 按照上面规范生成的 Markdown 条目，插入到合适位置并保存。

示例调用（文本形式）：
- question: "Spark 中 BroadcastHashJoin 的触发条件有哪些？如何诊断和优化？"
- category: "Spark"
- format_requirements: "需要 SQL 示例与排查建议"

示例输出（片段）：
## 1. Spark 面试题

### 1.2 BroadcastHashJoin 的触发与优化

（详细内容，保持与文档风格一致，含示例与扩展）

迭代与澄清：
- 如果问题模糊，我会返回一个简短澄清问题（例如：是否需要代码示例、目标语言等）。
- 如果用户希望我直接编辑文件，模版会自动执行追加并返回文件链接。

注意事项：
- 严格使用英文括号 ()。
- 保持文体与现有文档一致。

---

保存位置：工作区根目录 `\.prompt.md`。

如果你确认，我将从现在起使用这个 prompt 模板来处理你接下来提出的问题，并把问题与答案依文档风格追加到 [doc/大数据面试题.md](doc/大数据面试题.md)。
