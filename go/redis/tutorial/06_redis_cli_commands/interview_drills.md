# Redis CLI 面试演练题

## 1. 为什么线上不建议用 KEYS *？

期望回答：
- `KEYS` 是 O(N) 全量扫描，阻塞线程。
- 高并发下会拖慢整个 Redis 实例。
- 替代方案：`SCAN` 分批迭代。

## 2. 事务里某条命令失败会回滚吗？

期望回答：
- 不会自动回滚。
- Redis 事务是批量顺序执行模型。
- 若要复杂原子逻辑，优先 Lua。

## 3. String 与 Hash 如何选型？

期望回答：
- String：整对象缓存、计数器。
- Hash：字段级读写，减少序列化成本。
- 结合数据大小与访问模式决定。

## 4. 如何实现安全分布式锁？

期望回答：
- 加锁：`SET key value NX PX ttl`。
- 解锁：Lua 比对 value 后删除。
- 补充：锁续期、时钟漂移与主从一致性风险。

## 5. 如何排查 Redis 性能抖动？

期望回答：
- `INFO`, `SLOWLOG`, `LATENCY DOCTOR`, `MEMORY STATS`。
- 检查大 key、慢命令、淘汰次数、连接数。
- 结合业务高峰和命令画像分析。

## 6. 为什么排行榜通常用 ZSet？

期望回答：
- 自动按 score 排序。
- 支持高效范围查询和名次查询。
- 命令直观：`ZADD`、`ZREVRANGE`、`ZREVRANK`。

## 7. Stream 比 List 做队列好在哪里？

期望回答：
- 支持消费组、多消费者协作。
- 支持 ACK 与待处理消息追踪。
- 能回溯消费，更适合可靠消息。

## 8. 过期删除与淘汰策略有什么区别？

期望回答：
- 过期删除针对已设置 TTL 的 key。
- 淘汰策略在内存不足时触发。
- 二者目标不同，但共同影响命中率与稳定性。

## 9. 你会如何描述一次完整缓存读流程？

期望回答：
- 先查缓存，命中直接返回。
- miss 查数据库并回填。
- 配置过期策略并考虑击穿/雪崩保护。

## 10. 面试最后可反问什么？

建议反问：
- 当前 Redis 部署形态（单机/哨兵/集群）？
- 慢查询与大 key 如何监控治理？
- 热点 key 方案和故障演练机制是什么？
