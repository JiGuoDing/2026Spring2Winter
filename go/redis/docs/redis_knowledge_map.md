# Redis 核心知识图谱 (面试速查)

## 1. 数据类型与典型场景

- String：缓存对象、计数器、分布式锁 token。
- Hash：对象字段存储，便于局部更新。
- List：简单队列、时间线。
- Set：去重、标签、交并差。
- ZSet：排行榜、延迟队列。
- Stream：可靠消息流、消费组。
- Bitmap / HyperLogLog / GEO：位图统计、UV 估算、LBS 检索。

## 2. 持久化

- RDB：快照，恢复快，可能丢失最近一段数据。
- AOF：命令日志，数据更完整，文件更大。
- 混合持久化：重启快且数据损失较小（生产常用）。

## 3. 过期与淘汰

- 过期删除：惰性删除 + 定期删除。
- 内存淘汰策略：`noeviction`、`allkeys-lru`、`allkeys-lfu`、`volatile-ttl` 等。

## 4. 高可用与扩展

- 主从复制：读写分离与故障恢复基础。
- Sentinel：自动故障转移。
- Cluster：16384 槽分片，去中心化，支持水平扩容。

## 5. 高并发常见问题

- 缓存穿透：布隆过滤器 / 空值缓存。
- 缓存击穿：热点 key 互斥锁 / 永不过期 + 异步刷新。
- 缓存雪崩：过期时间随机化 + 限流降级。

## 6. 事务与原子性

- `MULTI/EXEC`：批量顺序执行，不支持传统回滚。
- Lua：Redis 内部单线程执行，天然原子，适合复杂读改写。

## 7. 面试高频误区

- Redis 单线程指的是命令执行线程，不代表网络 IO 完全单线程。
- 事务不是关系型数据库事务，不保证失败回滚。
- 分布式锁不能只写 `SETNX`，还需要过期时间和安全释放逻辑。
